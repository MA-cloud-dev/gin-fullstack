## 核心技术实现

### 1. SQLite 初始化切换

- 后端 `server/config.yaml` 默认数据库类型已改为 `sqlite`。
- 为了保留初始化页面，默认配置没有直接写死 `db-name`，首次启动仍会进入初始化流程。
- 前端初始化页默认选中 SQLite，`dbPath` 默认设置为当前 `server` 目录下的 `.`，初始化后会生成 `gva.db`。

### 2. 用户最后登录信息回写

- 在后端用户模型 `SysUser` 中新增：
  - `lastLoginIp`
  - `lastLoginAt`
- 依赖 Gorm 自动迁移，将字段同步到 `sys_users` 表。
- 用户登录成功后，在签发 token 的流程中同步回写当前登录 IP 和登录时间。
- 登录失败不会更新这两个字段，失败信息仍保留在系统登录日志表中。

### 3. 用户管理页展示

- 在用户管理列表页新增“登录IP”和“登录时间”两列。
- 登录时间按 `YYYY-MM-DD HH:mm` 格式显示。
- 若用户尚未登录过，则显示为 `-`，避免页面报错。

## 启动方式

### 前端启动

在项目根目录进入 `web` 后执行：

```bash
npm install
npm run dev
```

### 后端启动

在项目根目录进入 `server` 后执行：

```bash
go mod tidy
go run main.go
```

## 演示建议流程

1. 启动前后端服务。
2. 首次访问系统，进入初始化页面，确认数据库类型默认为 SQLite。
3. 完成初始化后，使用管理员账号登录系统。
4. 打开“用户管理”，新增用户 `a`。
5. 使用第二个浏览器登录用户 `a`。
6. 返回管理员浏览器刷新用户管理列表，查看用户 `a` 的登录 IP 和登录时间。
